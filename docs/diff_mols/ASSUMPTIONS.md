# diff_mols — Living Assumptions

Each row mirrors spec §6 with the question "how do I change this safely?" baked in.
Update this file in the same commit as the change.

| # | Assumption | Rationale | Current value | How to change | Impact if wrong |
|---|---|---|---|---|---|
| 1 | AVAS patterns correctly select the chemistry-relevant active space for metal complexes | Standard practice (`Cr 3d`, `Fe 3d`, `S 3p`); the builder writes `scf_summary.json` with the AVAS-selected MO energies + occupations for human inspection before launching the method matrix | Per molecule in `configs/<mol>.yaml::active_space.avas_patterns` | Edit YAML, re-run `python -m FRASCI.diff_mols.run --mol <slug> --build-integrals-only`, inspect `Outputs/diff_mols/<mol>/integrals/<geom>/scf_summary.json` | Wrong AS biases every method; never run the full matrix until AS looks right |
| 2 | Fragmentation by IAO populations gives clean per-atom / per-metal-center groupings | IAO is the standard for this in mrh + pyscf; tested on H₄ smoke | `auto_per_atom` / `auto_per_metal` in `fragmentation.py` | Swap to Boys / Pipek-Mezey in `fragmentation.py` if populations look smeared | Fragments overlap → LASSCF still works but "chem" partition becomes meaningless; fall back to h1diag in YAML |
| 3 | LASSIS `n_spin` defaults: 0 for closed-shell organics; 1 for Cr₂ (HS→BS singlet); 2 for Fe₂ / Fe₂S₂ (d⁵-d⁵ AFM) | Standard magnetic-coupling LASSIS practice | YAML `methods.lassis.n_spin` per molecule | Edit YAML, re-run lassis only | Wrong `n_spin` misses the relevant BS state; J / spin-gap numbers wrong (sometimes by sign) |
| 4 | Reference geometries for the 7 molecules are user-provided literature-optimized XYZs (organic molecules' geometries inline in YAML; metal complexes need files dropped into `configs/geometries/`) | Geometry optimization is out of scope | Inline XYZ for organics; `configs/geometries/<mol>.xyz` for metals (placeholder stubs until user replaces) | Edit YAML / replace XYZ; re-run `--build-integrals-only` then methods | Energies shift but relative comparisons (J, spin gaps) survive for reasonable geometries |
| 5 | CASSCF orbital opt enabled for metal complexes, off for organics | CASSCF improves d-orbital quality; unnecessary for π-only N=N | YAML `active_space.casscf` per molecule | Toggle in YAML | If off for a complex, AVAS orbitals may not be variationally optimal; methods still run but lower-quality reference |
| 6 | Inline FCI reference feasible up to active space (8, 8) | PySCF FCI direct_spin1 is sub-second at (8,8); ~minutes at (10,10) | `reference.computed_inline: true` only on small-AS configs | Set `false` in YAML for larger AS; provide external reference number | Without a reference, plots show absolute energies only |
| 7 | The 9-point coarse PES (1.00 / 1.10 / 1.25 / 1.50 / 1.75 / 2.00 / 2.25 / 2.50 / 3.00 Å) captures equilibrium + asymptote + avoided crossing | Standard recipe for N=N | YAML `geometry.scan_points` | Bump to 21-point medium scan once the per-mol bump banner goes green | Coarse misses fine structure near 1.5–2.0 Å — bump path addresses this |
| 8 | The 5-method ablation cleanly separates orbital-opt / fragmentation / multi-state contributions | Each method differs from another by exactly one of {COO, LASSCF, LASSIS} | Hard-coded method set in `METHOD_ORDER` | Add a new method as a new adapter in `methods.py` + new YAML field | If ablation is muddled, interpretation gets harder but raw numbers stay valid |
| 9 | The existing runners are callable as Python functions (no subprocess) | All `run_lasscf_*` and `run_lassi_lassis` expose `def run(...)`; `coo_adapter` exposes `run_baseline` and `run_outer_loop` | Direct imports in `methods.py` | If a runner needs a hook, add it in the same commit; Fe4S4 regression lock must still pass | A runner crashes inside `MoleculeBenchmark` → stack trace preserved (this is *why* we don't subprocess) |
| 10 | Basis defaults: `def2-SVP` for metal complexes, `cc-pVDZ` for organics | Light, well-tested basis sets that work in PySCF without ECP gymnastics | YAML `electronic_structure.basis` | Edit YAML, rerun everything | Larger basis is more accurate but slower; OK for warm-up benchmark |
| 11 | TrimCI threshold 0.01, max_dets 2000, coo_cycles 2 defaults align with cyc=2 production for Fe4S4 | Documented in `project_lasscf_reorg_20260622` | YAML `methods.*` per molecule | Edit per-method block in YAML | Tighter threshold + more dets → lower energy but slower |
| 12 | Boys localization is applied when `active_space.boys: true` explicitly, OR when auto fragmentation is detected (`auto_per_atom` / `auto_per_metal` / `chem_bond` in the fragmentation list) | Boys frame ensures FCIDUMP integrals, mo_coeff.npz, and IAO population analysis all live in the same localized orbital frame; prevents basis mismatch in downstream fragmentation. Energy-sorted Aufbau seeding (diagonalizing h1 in the Boys frame) ensures dets.npz row 0 (reference determinant) has the correct occupations in the Boys frame. | YAML `active_space.boys` or auto fragmentation detection in `integrals_builder.py::_AUTO_FRAG_KINDS` | Set `boys: true` in YAML for explicit Boys; switch `fragmentation.py` from `auto_per_atom` to other strategies if Boys behavior is unwanted | Boys-frame mismatch → dets.npz row 0 has wrong occupations, LASSCF determinant seeding fails silently (uses HF instead), fragment electron counting is incorrect |
| 13 | `chem_bond` partition (literature-style bond-grouped fragmentation) is preferred over `auto_per_atom` for organics with localized multiple bonds (N=N, C=C). Atom groups follow Hermes & Gagliardi 2019 (JCTC) fragmentation of azomethane and C2H6N4: keep each multiple bond inside one fragment with the spectator alkyl/H groups as separate fragments | LASSCF is accurate when strong correlation is **intra-fragment** and inter-fragment correlation is weak. Splitting an N=N across a fragment boundary (which `auto_per_atom` does when the two N's get separate fragments) violates this assumption and gives 200+ mHa errors on diazene (see progress log 2026-06-23). `chem_bond` reproduces the literature partition: me2n2 → [CH3 on N0]·[N=N]·[CH3 on N1], C2H6N4 → [HN=N-left]·[bridge]·[N=NH-right]. Verified on me2n2: (4,4) lands on N=N as in the paper. | `configs/<mol>.yaml::fragmentation` entry `{name: chem_bond, orbital_lists: chem_bond, atom_groups: [[...], [...], ...]}`; partition name added to `methods.lasscf_*.partitions:` lists. Currently enabled for `me2n2` and `c2h6n4_tetrazene`; **skipped for diazene_cis / diazene_trans** because for isolated HN=NH the literature partition is 1 fragment = trivial CASSCF — no fragmentation lesson. | Edit `atom_groups` in YAML to reflect a different chemical partition; rerun lasscf methods. The resolver enforces disjoint, complete atom coverage (validation in `fragmentation.py::_validate_atom_groups`). | If `atom_groups` puts a multiple bond across a fragment boundary, LASSCF degrades like `auto_per_atom`. Empty groups (no MOs assigned by IAO) are silently dropped and noted in the partition description. |
